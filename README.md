# Alpha Screener

Alpha Screener 是一个美股命令行筛选器。运行 `asc` 后，它会更新行情、输出未来第 14 个交易日最值得关注的爆发候选、汇总最近 30 个成熟决策日的历史模拟，并在结果可知之前保存当天完整排名。

## 安装

需要 Python 3.11–3.14 和 [uv](https://docs.astral.sh/uv/)。在项目目录执行：

```bash
uv tool install .
```

已有旧版时执行 `uv tool install --force .`；项目开发仍建议使用下方的 editable 安装，避免本地同版本 wheel 缓存。

开发时需要让命令实时反映源码改动，可使用：

```bash
uv tool install --editable .
```

项目不需要 API Key。首次运行需要下载数千只股票的行情，耗时取决于网络和 Yahoo Finance 的可用性。

## 使用

```bash
asc                 # 显示默认 Top-10
asc --top 20        # 显示 Top-20
asc backtest         # 详细显示最近 30 个成熟决策日
asc backtest --days 45
```

`--top` 只改变终端显示数量；账本始终保存当日完整合格股票池。`--days` 只能为 1–45，默认 30。

## 数据和失败行为

- 股票目录来自 Nasdaq Trader 的 `nasdaqlisted.txt` 和 `otherlisted.txt`，过滤测试证券、ETF 和明显的非股票工具；SPY 只作为市场基准，不会作为候选输出。
- 最终候选会用 SEC EDGAR 做时点检查。只有文件能高置信确认候选本身是已签署确定性交易的目标方时才排除；收购方 8-K、发行人自要约、传闻和非约束报价不会仅凭表单被误排除。SEC 状态不完整时不生成新排名。
- SEC 请求会限速并缓存到 `~/.alphascreener/data/sec/`。如需覆盖默认标识，可设置包含组织名和联系信息的 `ALPHASCREENER_SEC_USER_AGENT`。
- 日线 OHLCV 来自 Yahoo Finance；收益与比例信号使用复权 Close，`$5` 价格门槛和平均成交额使用当时原始 Close，避免后续拆股或分红回溯改变历史预筛。
- 交易日统一使用稳定的 NYSE（XNYS）交易所日历；SPY 用作基准并校验窗口完整性，不再充当日历来源。
- Nasdaq 目录会另外保存带获取时间、内容哈希和原始文件哈希的不可变快照，供此后研究按 `known_at` 重放。快照创建以前的多年研究仍是当前存续股票池诊断，不能倒推出历史股票池。
- 行情按 `ticker + dt` 合并并原子替换分区；部分批次失败不会删除其他股票的历史数据。
- 行情必须包含有限的正价格和非负成交量；非法数据不会进入股票池。
- 任一官方股票目录不可用或为空时，`asc` 会停止，不会用半个市场生成排名。
- 刷新覆盖率低于 90% 时，`asc` 不会输出或记录新排名。
- SPY 必须具备决策日数据和完整 60 日窗口，否则 `asc` 不会生成排名。
- 本地 SPY 行情必须至少更新到最近一个已经收盘的 NYSE 交易日；刷新失败时不会拿旧缓存生成新账本。
- 预测账本中的标的若已离开当前目录，只要确切第 14 日结果仍未落盘，后续 `asc` 仍会继续补取；它们不会混入今日股票池或覆盖率。
- 行情分区和预测账本使用进程级写锁，多个 `asc` 进程不会互相覆盖提交。
- 行情保存在 `~/.alphascreener/data/ohlcv/`，不可变预测账本保存在 `~/.alphascreener/predictions/`。历史回测按需重算，不另存一份派生记录。

## 预测与评估契约

- 输入窗口：最多 60 个交易日。
- 预测窗口：未来第 14 个交易日的收盘收益。
- 爆发事件：收益同时达到 15% 和当日合格股票池第 95 百分位，以两者中更高者为准。
- 默认输出：Top-10。
- 动态预筛：每个决策日只使用当时可见的数据，要求连续 60 个市场日、价格不低于 5 美元、20 日平均成交额不低于 1,000 万美元、决策日前 20 日成交额中位数不低于 500 万美元且至少 18 日有有效价格与成交量，再只保留当日流动性最高的 2,000 只股票。
- 结构性事件：只检查爆发初排 Top-30；排除一只后会继续检查补入者，保证最终 Top-30 都有明确 SEC 状态。历史回测使用对应历史收盘时点，包含 NYSE 提前收市。
- 风险标签：第 14 日收益不高于 -10% 为严重下跌，不高于 -20% 为灾难损失；未来 14 日最低收盘相对决策日不高于 -15% 作为路径风险诊断。
- 自动回测：每次 `asc` 汇总最近 30 个已经成熟的历史决策日；`asc backtest` 逐日显示 1–45 日详情。价格、成交量和特征只读取各决策日及此前最多 60 个市场日。该回测使用当前股票目录，带有生存者偏差，明确标记为 `CURRENT_UNIVERSE_DIAGNOSTIC`。
- 风险回测：逐日报告 `Downside@10`、`CatLoss@10`、路径风险与 Top-10 等权第 14 日收益；汇总的 `ES10` 是最差 10% 有效日期的平均组合收益。
- 结果覆盖：一个决策日至少要有 90% 的股票获得未来收益，且原始 Top-10
  结果完整，才进入回测或预测评估；缺失结果不会由后续名次补位。
- 回测范围内只要有一个日期无效，整体状态即为 `INCONCLUSIVE`；汇总指标仅描述其余有效日期。
- 单日达标：`Precision@10 >= 10%`，并且严格高于同日完整股票池基础爆发率。

历史回测是“今天用当前策略重放过去”，可以随时计算；预测账本是“结果未知时实际保存的答案”，只包含真正运行过 `asc` 的日期。漏跑不会影响当前筛选或历史回测，但不能事后补成前瞻证据。`asc` 会自动评价已经成熟的账本记录，不再需要单独的 `evaluate` 命令。

评分信号、权重、股票池规则或标签契约发生变化时，必须更新 `STRATEGY_VERSION`，避免新旧结果混合。

一个策略版本只有连续 5 个真实成熟预测日逐日达标，才算达到目标；回测记录不能替代这 5 个事前预测日。

## 开发验证

```bash
uv sync --extra dev
./scripts/dev-plan.sh
```

模型研究不属于公开 `asc` 接口。需要复现冻结的长周期资格验证时执行：

```bash
uv sync --extra dev --extra research
uv run python -m alphascreener.research --backfill-start 2020-01-01
```

该命令使用相同的逐日预筛股票池，对当前 `rank-v6` 核心评分、正则化线性基线和风险护栏排序集成做严格时间顺序比较。LambdaMART 用三级相关性保持爆发为一级目标、严重下跌为二级约束；它与当前稳健排名只在固定权重网格中融合，并先要求验证期三项尾部指标不退化。最后由只在冻结的当前 `rank-v6` Top-30 条件分布上训练的独立下行模型做最多 3 位的局部重排，避免用同一训练样本的挑战模型名次选择风险训练池。训练标签必须在对应 `model_as_of` 前成熟，训练与验证之间隔离 14 个交易日，最后 252 个有效日只做走步样本外测试；报告使用 14 日成对区块自助法，并在每次重采样中重新计算尾部风险和 ES，同时检查四个连续测试块与 SPY 涨跌状态。研究行情按 63 个决策日分块计算，不写预测账本。

内部研究还会加载无需密钥的官方数据：SEC 季度 `master.zip` 申报索引、SEC Form 3/4/5 结构化数据集和 FINRA 半月空头持仓文件。原始响应按观察日期和内容哈希不可变缓存到 `~/.alphascreener/data/official-research/raw/`，申报、发布日期、结算日和本地首次观察日分别保留。SEC 季度索引没有可直接使用的精确受理时刻，FINRA 文件也没有可依赖的日内发布时间，因此两者都延迟到下一个 XNYS 决策日使用；修订文件不会倒填到首次观察以前。免费 FINRA 全市场历史从 2021-06 开始，更早日期和跨越边界的 60 日窗口以缺失或部分覆盖表示，不会伪装成零信号。

挑战策略只有同时满足 P@10 至少提高 2 个百分点、成对置信区间下界大于 0、最近 45 日和各分段不退化，并且严重下跌、灾难损失和 ES 均不恶化且至少一项实质改善，才允许创建新的生产策略版本。未通过时 `asc` 会继续使用当前已冻结策略，不会把研究结果偷偷用于当天排序。

OHLCV 风险挑战模型在 252 个锁定测试日取得 `P@10=14.33%`，低于 `rank-v6` 的 `14.44%`；虽然 `Downside@10`、`CatLoss@10` 和 `ES10` 分别从 `24.84%`、`12.14%`、`-18.81%` 改善到 `23.97%`、`11.67%`、`-18.04%`，但爆发提升的成对 95% 区间为 `[-0.52, +0.28]` 个百分点，因此判定为 `REJECT`。

在完全相同的 5,889 只当前股票、2,930,713 个研究样本和 252 个锁定测试日上，加入 20 个 SEC/FINRA 特征后的挑战模型取得 `P@10=14.09%`、`Downside@10=24.37%`、`CatLoss@10=11.87%`、`ES10=-18.39%`；锁定期申报、内部人交易和空头覆盖率分别为 `99.97%`、`99.58%`、`98.93%`。相对 `rank-v6` 的 P@10 下降 `0.36` 个百分点，成对 95% 区间为 `[-0.75, +0.04]`，最近 45 日也下降 `0.22` 个百分点，因此同样 `REJECT`。该特征族是在当前锁定期已被观察后提出的，本次结果只属于探索性消融，不能作为晋级证据。

生产中的 `rank-v7-guardrails` 继续保留 `rank-v6` 爆发评分，只增加稳定流动性预筛、确定性交易排除和完整风险回测；不会下载这些研究数据，也不展示未经资格验证的个股风险分数。

多年历史仍主要来自本次运行取得的当前官方股票目录，SEC 的 `company_tickers.json` 也是当前 ticker 与 CIK 的映射且不保证历史完整性，因此报告明确标记为 `CURRENT_SURVIVOR_UNIVERSE_RESEARCH_DIAGNOSTIC`，包含幸存者偏差和历史代码映射限制。即使挑战模型通过离线门槛，也只说明它有资格进入新的事前预测账本验证，不能替代连续 5 个真实成熟预测日。

CI 会在 Python 3.11、3.12、3.13 和 3.14 上运行 lint 与完整测试。

本工具输出的是研究候选，不构成投资建议。外部数据可能延迟、缺失或受供应商限制。
