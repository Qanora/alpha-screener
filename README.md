# Alpha Screener

Alpha Screener 是一个美股命令行筛选器。运行 `asc` 后，它会获取最近约 60 个交易日的可用行情，输出未来 14 个交易日最值得关注的爆发候选，并在结果可知之前保存完整排名。

## 安装

需要 Python 3.11–3.14 和 [uv](https://docs.astral.sh/uv/)。在项目目录执行：

```bash
uv tool install .
```

开发时需要让命令实时反映源码改动，可使用：

```bash
uv tool install --editable .
```

项目不需要 API Key。首次运行需要下载数千只股票的行情，耗时取决于网络和 Yahoo Finance 的可用性。

## 使用

```bash
asc                 # 显示默认 Top-10
asc --top 20        # 显示 Top-20
asc evaluate        # 评估已经成熟的 14 交易日预测
```

`--top` 只改变终端显示数量；账本始终保存当日完整合格股票池，确保后续能够计算真实的基础命中率和 `lift@K`。

## 数据和失败行为

- 股票目录来自 Nasdaq Trader 的 `nasdaqlisted.txt` 和 `otherlisted.txt`，过滤测试证券、ETF 和明显的非股票工具；SPY 只作为市场基准，不会作为候选输出。
- 日线 OHLCV 来自 Yahoo Finance，使用自动复权价格。
- 行情按 `ticker + dt` 合并并原子替换分区；部分批次失败不会删除其他股票的历史数据。
- 行情必须包含有限的正价格和非负成交量；非法数据不会进入股票池。
- 任一官方股票目录不可用或为空时，`asc` 会停止，不会用半个市场生成排名。
- 刷新覆盖率低于 90% 时，`asc` 不会输出或记录新排名。
- SPY 必须具备决策日数据和完整 60 日窗口，否则 `asc` 不会生成排名。
- 行情分区和预测账本使用进程级写锁，多个 `asc` 进程不会互相覆盖提交。
- 行情保存在 `~/.alphascreener/data/ohlcv/`，预测账本保存在 `~/.alphascreener/predictions/`。
- 旧三列账本仍可读取，但因缺少完整股票池大小，不参与可信的 lift 计算。

## 预测与评估契约

- 输入窗口：最多 60 个交易日。
- 预测窗口：未来第 14 个交易日的收盘收益。
- 爆发事件：收益同时达到 15% 和当日合格股票池第 95 百分位，以两者中更高者为准。
- 默认输出：Top-10。
- 结果覆盖：一个预测日期至少要有 90% 的股票获得未来收益，且原始 Top-K
  结果完整，才进入评估；缺失结果不会由后续名次补位。
- 置信区间：至少积累 20 个成熟预测日期后，才输出确定性区块 bootstrap 95% 区间。

评分信号、权重、股票池规则或标签契约发生变化时，必须更新 `STRATEGY_VERSION`，避免新旧结果混合。

一个策略版本只有同时满足以下条件，才可作为有效改进：

1. 至少 20 个成熟预测日期，平均结果覆盖率不低于 90%。
2. `lift@10 > 1`。
3. Top-10 平均 14 日收益大于 0。
4. Precision@10 的 95% 置信区间下界高于同期完整股票池基础爆发率。
