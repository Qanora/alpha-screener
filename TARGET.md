# 目标

## 核心结果

- 面向需要研究美股短期机会的用户。用户运行唯一入口 `asc`，获得在决策时点看来、未来 14 个交易日内最可能出现爆发性上涨的美股候选排名。
- 排名质量必须能够用结果未知时保存的预测和后来真实发生的行情验证，不能以主观观感、样例输出或回看后调整代替。

## 必需功能

- `asc` 自动取得当前完整的美国交易所上市普通股与 ADR 范围，以及截至决策日可获得的最近 60 个交易日行情；不得读取决策日之后的信息。
- `asc` 排除历史不足、行情过期、价格或流动性不满足交易要求的标的，输出默认 Top-10 排名；市场基准或股票池不完整时不生成结果。
- `asc` 在输出当前候选前，使用同一策略完成最近至少 3 个成熟决策日的逐日走步回测。每个回测只能读取该决策日前最多 60 个交易日，并以第 14 个后续交易日的真实结果评价；不足 3 个有效回测日时不输出当前候选。
- `asc` 在结果可知之前保存决策日、策略版本、完整合格股票池、排名和分数。同一决策日与策略版本的记录不可被覆盖。
- `asc evaluate` 在第 14 个后续交易日结果成熟后，按策略版本和决策日报告 Precision@10、完整股票池基础爆发率、结果覆盖率以及当日是否达标；缺失的 Top-10 结果不得由后续名次补位。
- “爆发”定义为第 14 个后续交易日的复权收盘收益，同时达到 15% 和当日合格股票池收益第 95 百分位，以两者中更高者为阈值。

## 输入与输出

- 可信输入：完整且可识别证券类型的当前美国上市目录；截至决策时点的日频复权价格、成交量和市场基准数据。输入来源可以替换，但必须能检测缺失、过期、非法值和不完整覆盖。
- 正确输出：带有明确决策日期的有序候选列表。每个候选至少包含股票代码、名次和可比较分数；默认展示 10 个，完整排名必须先于未来结果持久化。
- 必要副作用：只在用户本地保存再次运行所需的行情数据、与事前预测账本分离的回测记录，以及不可变预测记录；并发运行不得损坏、丢失或覆盖已提交数据。
- 失败输出：外部数据不可用、股票池不完整、覆盖不足或市场基准无效时，以可理解的信息结束，不输出或记录可能误导的新排名。

## 明确不做

- 不执行交易，不管理账户、订单、仓位、组合或风险预算。
- 不提供网页、服务端 API、图形界面、通知、社交或多用户能力。
- 不筛选非美股资产，不预测 14 个交易日以外的目标，也不承诺收益。
- 不为了兼容旧命令、旧数据格式、旧策略或历史实现保留额外路径；只有当前目标所需的 `asc` 行为和可验证预测记录属于项目范围。
- 不把事后计算的回测记录冒充事前预测；回测只能作为每次输出的最低历史证据，不能替代连续 5 个真实成熟预测日的终态标准。
- 不把终端装饰、开发流程或测试通过本身当作预测目标已经实现的证据。

## 运行完成标准

- 从无项目环境和无本地数据状态开始，执行 `uv tool install .` 能安装 `asc`；运行 `asc` 会自动准备数据并完成一次真实筛选，不要求用户预先调用其他命令或提供密钥。
- 在外部数据完整时，`asc` 正常退出并显示决策日、最近至少 3 个成熟决策日的逐日回测记录和 Top-10 候选，同时写入该日该策略版本的完整排名；当天再次运行不会覆盖已保存预测。
- 每条回测记录至少包含策略版本、历史决策日、结果日、完整股票池大小、结果覆盖率、Precision@10 和基础爆发率。回测使用本次运行取得的当前官方股票目录并明确标记为当前股票池回测；价格、成交量、特征和排名必须由历史决策时点可获得的数据独立重算，不能读取后续行情。
- 在股票目录、行情覆盖或市场基准不满足要求时，`asc` 正常给出失败原因，不显示候选、不写入新预测，也不遗留临时提交文件。
- 准确率定义为逐日 `Precision@10 = Top-10 中实际爆发的标的数 / 10`。一个决策日只有在完整股票池结果覆盖率不低于 90%、原始 Top-10 结果完整、`Precision@10 >= 10%` 且严格高于当日完整股票池基础爆发率时才算达标。
- 对连续 5 个已经成熟的真实决策日运行 `asc evaluate`；只有 5 日逐日全部达标，当前策略才算达到目标。某一天的额外命中不能抵消其他日期未达标。
- 真实运行不得出现未处理异常。允许输出下载进度、覆盖率和明确的可恢复警告；允许留下行情缓存、不可变预测账本和进程锁，不允许留下损坏分区、未完成临时文件或在失败后产生的新预测。
